I'd like to have a list of examples in which it was a sick reasoning-move to make a vague notion precise.

e.g.: cell, reference frame, wavefunction?, atom?, some stuff from algebraic geometry?, continuous function?, preference as vNM?

Like, why is it surprisingly often a good idea to start from something vague, and then to try to construct something concrete that roughly 'fits in the same place' in reasoning? Why not just create concepts from scratch? Why should we expect there to be anything concrete that an initial vague concept is related to, as opposed to it just not making sense? 

I guess maybe the answer is that this is just what a default way to create concepts from scratch looks like! Perhaps when we create a concrete concept from scratch, it is natural to start from a list of desiderata, or maybe (in particular) a set of cases/objects that we want to have as instances of this desired concept (note that we can view each one as providing a desideratum: the desideratum that it is an instance of the new concept). And having this list of desiderata is a lot like having a vague concept! Maybe this kind of thing is indeed a/the natural format for concepts in our heads? For instance, when doing algebraic geometry, I'd guess that a trained algebraic geometer has a bunch of stuff they know about schemes, they have a sense for schemes that is well-approximated as a (likely structured) set of facts about schemes — more precisely, facts about how particular schemes relate to other schemes and other other objects, and how schemehood relates to other things (in this view, a definition of a scheme would by default (maybe when we are examining only how it is mentally represented/stored) just be one fact about schemes among many, though it might have the additional special property that all facts about schemes could be derived from it).

Ok, but still, when we want to expand our net of concepts, to enrich it with a new concept, (so that we can see further, we can prove more,) why ought we to start from a given proto-concrete-concept, and modify it, as opposed to starting anew? We could list a new set of constraints, and then look for a concept satisfying it. How does this process of vaguely getting at a real missing concept work? How come there really is often something there? Maybe we've just listed a bunch of properties that are not obviously contradictory, and there then often indeed is some formal object in some formal system with these properties?



What are some examples of concept-creation that do not fit this mold very well?
