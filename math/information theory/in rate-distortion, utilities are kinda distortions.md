
one can worry: in the rate-distortion theorem, isn't it silly to call the distortion function a 'distortion function', as opposed to calling it a utility function, as it can be any function of the input alphabet and a potentially completely different output alphabet? i think it is indeed sorta silly, but not that silly. the reason it isn't that silly is that any utility function can be viewed as a distortion function on some function of the input. this is because we can just list for each input x the possible outputs y ranked by utilities from highest to lowest. the highest utility y value is a function of x. we can shift the utility function on this x alone by any const we like without changing the ranking of coding schemas, so without really changing the utility maximization problem. let's shift it so that the top y for this x has utility 0. do this for every value x. and now the coding problem is just to recover some function of the input with a distortion loss! (note of course this distortion loss isn't a metric or even symmetric or whatever, and i guess probably there won't be a way to reduce to that case)
