
* note that in math, we didn't straightforwardly turn existing proofs into formal proofs (nor existing statements into formal statements)
	* as in, we didn't do anything close to word-by-word translation. we didn't even do sentence-by-sentence translation 
* instead, we created a language in which formal proofs can be written. there is a way to turn informal proofs into formal proofs, but it is actually fairly complicated. for instance, we needed to figure out how to specify all the objects we're using in set theory
* analogously, to create a protocol for physics olympiad solution verification, we needn't be staying that close to existing physics olympiad solutions
	* it is eg totally fine if an existing solution doesn't pass as a proper solution for one of the following reasons:
		* it leaves out very many things (this is also true for math olympiad solutions)
		* it says some wrong stuff (most long math papers probably make some wrong claims)
	* it could also be fine if some sentences in physics olympiad solutions cannot really be turned into "proper sentences" that fit in our schema. in math, we probably have lots of broken ideas that ultimately need to be said quite differently as well (eg some idea ends up not really making sense, or it ends up being tricky to make sense of it)
* still, it obviously makes sense to actually look at physics olympiad solutions, to try to identify some nice thing that is shadowed in them
