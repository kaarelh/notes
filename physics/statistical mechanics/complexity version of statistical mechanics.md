
Instead of having a list of macrostates given by fiat, have a complexity metric on all macrostates, i.e. just all 2^n sets of microstates. This could be something like Solomonoff prior, i.e. shortest program checking membership in the macrostate, with primitive queries being to positions and momentums of particles. It would be interesting to then prove something like "Take a small simple macrostate at time $t$. Time-evolve all its microstates to time $t+1$. Then look at all macrostates containing its image at time $t+1$. Claim: There is a tradeoff between complexity of each of these macrostates and how tightly it contains the image. I.e. we either take the image directly which has high complexity, or some low complexity state which is much bigger. (Or maybe this is the case for most microstates at time $t$, at least. I'd guess there is some statement like this from the usual treatment which could generalize/analogize to this version.)"

Maybe there is an issue here with one simplish way to describe the state at $t+1$ being that it's time-evolved from a particular simple macrostate at time $t$. Maybe time-evolution is just very hard to simulate? Maybe want to generally do faster than linear, e.g. some sublinear statistical query model (this intuitively seems right).
