Do searching for a valid comp path from input to accept state via midpoint search. the point is that even though there are exp f(n) possible states and any computation path could even include exp f(n) states in sequence, we look for a valid path without remembering the path using midpoint search. to check that there is a path of a certain length, you loop over all states as possible midpoints, searching for paths of half the length from start to proposed mid and from proposed mid to end. to keep this recursion state in mind, you don't need a full path — you just need a midpoint, and then a choice of another midpoint, and then another, for only log exp f(n) = f(n) total things of length f(n) each to keep in mind at once, for a total of f(n)^2 space
